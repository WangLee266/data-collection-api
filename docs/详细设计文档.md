# 详细设计文档

## 1. 系统概述

### 1.1 系统名称
开源数据采集系统 (Open Source Data Collection System)

### 1.2 系统目标
构建一个分布式的开源数据采集系统，支持从新闻网站、社交媒体、智库等多渠道采集数据，提供信源管理、任务调度、节点监控、数据检索等功能。

## 2. 系统模块设计

| 模块编号 | 模块名称 | 功能描述 |
|---------|---------|---------|

#### 3.4.1 模块职责
- 节点注册与发现
- 节点状态监控
- 资源使用统计

#### 3.4.2 节点状态监控

```
┌─────────────────────────────────────────────────────────────┐
│                    NodeHeartbeatMonitor                     │
├─────────────────────────────────────────────────────────────┤
│ 心跳机制:                                                    │
│   - 节点每10秒上报一次心跳                                    │
│   - 超过60秒无心跳判定为WARNING                               │
│   - 超过180秒无心跳判定为ERROR                                │
├─────────────────────────────────────────────────────────────┤
│ 资源监控:                                                    │
│   - CPU使用率                                                │
│   - 内存使用率                                               │
│   - 磁盘使用率                                               │
│   - 任务数量                                                 │
├─────────────────────────────────────────────────────────────┤
│ 预警规则:                                                    │
│   - CPU > 85%: WARNING                                       │
│   - Memory > 90%: WARNING                                    │
│   - Disk > 90%: CRITICAL                                     │
└─────────────────────────────────────────────────────────────┘
```

#### 3.4.3 负载均衡策略

```
┌─────────────────────────────────────────────────────────────┐
│                   LoadBalanceStrategy                       │
├─────────────────────────────────────────────────────────────┤
│ 策略1: 轮询 (RoundRobin)                                    │
│   - 按顺序依次分配任务                                        │
│                                                              │
│ 策略2: 最小连接数 (LeastConnection)                          │
│   - 选择当前任务数最少的节点                                  │
│                                                              │
│ 策略3: 加权轮询 (WeightedRoundRobin)                         │
│   - 根据节点性能设置权重                                      │
│   - 性能 = (100 - CPU%) * (100 - Memory%) / 10000           │
│                                                              │
│ 策略4: 地理位置优先 (GeoPriority)                            │
│   - 根据信源地理位置选择最近节点                              │
└─────────────────────────────────────────────────────────────┘
```

## 10. 预警监测模块设计 (M09)

### 10.1 模块概述

预警监测模块用于配置和执行预警监测任务，支持：
- 无数据超时预警
- 采集成功率监控
- 数据量异常波动检测
- 页面结构变化检测
- 账号限流检测

### 10.2 监测流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 创建监测任务  │───▶│ 选择监测目标  │───▶│ 配置预警规则  │
└──────────────┘    └──────────────┘    └──────────────┘
                                              │
                                              ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 任务执行中    │◀───│ 启动任务      │◀───│ 配置通知方式  │
└──────────────┘    └──────────────┘    └──────────────┘
       │
       ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 周期性检测    │───▶│ 触发预警      │───▶│ 发送通知      │
└──────────────┘    └──────────────┘    └──────────────┘
```

### 10.3 预警规则配置

| 规则类型 | 阈值配置 | 预警级别 |
|---------|---------|---------|
| 无数据超时 | 3小时/6小时/12小时/1天/3天 | 严重 |
| 成功率阈值 | <80% / <60% / <40% | 一般 |
| 数据量波动 | ±30% / ±50% / ±70% | 一般 |
| 页面结构变化 | 启用/禁用 | 严重 |
| 账号限流检测 | 启用/禁用 | 严重 |

### 10.4 监测频率选项

| 频率 | 说明 |
|-----|------|
| 15m | 每15分钟检测一次 |
| 30m | 每30分钟检测一次 |
| 1h | 每1小时检测一次 |
| 2h | 每2小时检测一次 |
| 6h | 每6小时检测一次 |
| 12h | 每12小时检测一次 |
| 24h | 每24小时检测一次 |
