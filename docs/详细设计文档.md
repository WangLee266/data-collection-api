# 详细设计文档

## 1. 系统概述

### 1.1 系统名称
开源数据采集系统 (Open Source Data Collection System)

### 1.2 系统目标
构建一个分布式的开源数据采集系统，支持从新闻网站、社交媒体、智库等多渠道采集数据，提供信源管理、任务调度、节点监控、数据检索等功能。

### 3.4 节点管理模块 (M04)

#### 3.4.1 模块职责
- 节点注册与发现
- 节点状态监控
- 资源使用统计

#### 3.4.2 节点状态监控

```
┌─────────────────────────────────────────────────────────────┐
│                    NodeHeartbeatMonitor                     │
├─────────────────────────────────────────────────────────────┤
│ 心跳机制:                                                    │
│   - 节点每10秒上报一次心跳                                    │
│   - 超过60秒无心跳判定为WARNING                               │
│   - 超过180秒无心跳判定为ERROR                                │
├─────────────────────────────────────────────────────────────┤
│ 资源监控:                                                    │
│   - CPU使用率                                                │
│   - 内存使用率                                               │
│   - 磁盘使用率                                               │
│   - 任务数量                                                 │
├─────────────────────────────────────────────────────────────┤
│ 预警规则:                                                    │
│   - CPU > 85%: WARNING                                       │
│   - Memory > 90%: WARNING                                    │
│   - Disk > 90%: CRITICAL                                     │
└─────────────────────────────────────────────────────────────┘
```

#### 3.4.3 负载均衡策略

```
┌─────────────────────────────────────────────────────────────┐
│                   LoadBalanceStrategy                       │
├─────────────────────────────────────────────────────────────┤
│ 策略1: 轮询 (RoundRobin)                                    │
│   - 按顺序依次分配任务                                        │
│                                                              │
│ 策略2: 最小连接数 (LeastConnection)                          │
│   - 选择当前任务数最少的节点                                  │
│                                                              │
│ 策略3: 加权轮询 (WeightedRoundRobin)                         │
│   - 根据节点性能设置权重                                      │
│   - 性能 = (100 - CPU%) * (100 - Memory%) / 10000           │
│                                                              │
│ 策略4: 地理位置优先 (GeoPriority)                            │
│   - 根据信源地理位置选择最近节点                              │
└─────────────────────────────────────────────────────────────┘
```
